# Documentation Organization Guide

This guide shows which documentation files should go in which category.

## How to Organize

You can move files manually or use these commands:

```bash
# Move getting started docs
mv QUICK_START.md docs/01-getting-started/
mv DEMO_CREDENTIALS.md docs/01-getting-started/
mv ENVIRONMENT_SETUP*.md docs/01-getting-started/

# Move architecture docs
mv ARCHITECTURE*.md docs/02-architecture/
mv SCHEMA_DESIGN.md docs/02-architecture/
mv *_ARCHITECTURE*.md docs/02-architecture/

# Move development docs
mv DEVELOPER*.md docs/03-development/
mv LINT*.md docs/03-development/
mv TYPESCRIPT*.md docs/03-development/
mv FLOATING_PROMISES_GUIDE.md docs/03-development/

# Move deployment docs
mv DEPLOYMENT*.md docs/04-deployment/
mv CI_CD*.md docs/04-deployment/
mv NETLIFY*.md docs/04-deployment/
mv PRODUCTION*.md docs/04-deployment/

# Move testing docs
mv TESTING*.md docs/05-testing/
mv TEST*.md docs/05-testing/
mv *_TEST*.md docs/05-testing/

# Move security docs
mv SECURITY*.md docs/06-security/
mv JWT*.md docs/06-security/
mv ED25519*.md docs/06-security/
mv ACCESS_MANAGEMENT*.md docs/06-security/

# Move archive docs (completed phases, old summaries)
mv PHASE_*.md docs/10-archive/
mv SESSION_SUMMARY*.md docs/10-archive/
mv *_COMPLETE.md docs/10-archive/
```

## Quick Organization Commands

### Move all at once (be careful!)
```bash
# Create backup first
cp -r . ../jala2-app-backup

# Move files by pattern
for file in PHASE_*.md; do mv "$file" docs/10-archive/ 2>/dev/null; done
for file in *_COMPLETE.md; do mv "$file" docs/10-archive/ 2>/dev/null; done
for file in SESSION_*.md; do mv "$file" docs/10-archive/ 2>/dev/null; done
for file in TYPESCRIPT_*.md; do mv "$file" docs/03-development/ 2>/dev/null; done
for file in DEPLOYMENT_*.md; do mv "$file" docs/04-deployment/ 2>/dev/null; done
for file in TESTING_*.md; do mv "$file" docs/05-testing/ 2>/dev/null; done
for file in SECURITY_*.md; do mv "$file" docs/06-security/ 2>/dev/null; done
for file in TEST_*.md; do mv "$file" docs/05-testing/ 2>/dev/null; done
```

## Manual Organization (Recommended)

It's better to organize manually to ensure files go to the right place:

1. Review each file
2. Determine its category
3. Move to appropriate folder
4. Update any broken links
5. Update category README if needed

---

**Note:** Some files may fit in multiple categories. Use your best judgment or create symbolic links if needed.
